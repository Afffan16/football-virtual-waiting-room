# Purpose

This document provides an estimated operational cost analysis for the Football Virtual Waiting Room.

The objective is to demonstrate that the proposed serverless architecture is not only scalable but also cost-efficient by leveraging AWS managed services and DynamoDB's consumption-based pricing model.

> **Disclaimer**
>
> The figures presented are estimates intended for architectural planning and comparison. Actual AWS costs depend on region, request volume, data size, concurrency, logging, and current AWS pricing. Refer to the official AWS Pricing pages before production deployment.

---

# Cost Drivers

The primary operational costs originate from:

- Amazon DynamoDB
- AWS Lambda
- Amazon API Gateway
- Amazon CloudWatch
- Data Transfer

Because the solution is entirely serverless, there are no always-running compute instances.

---

# DynamoDB

## Billing Mode

The solution uses:

```
On-Demand Capacity Mode
```

Reasons:

- Automatically scales
- No capacity planning
- Ideal for unpredictable traffic spikes
- Pay only for requests processed

# Cost Optimization Techniques

## On-Demand Billing

Avoids paying for idle capacity.

---

## TTL

Automatically deletes:

- Expired sessions
- Expired tokens

Reducing storage costs without scheduled cleanup jobs.

---

## Minimal GSIs

Only essential Global Secondary Indexes are created.

Benefits:

- Lower storage
- Lower write amplification
- Lower indexing costs

---

## Query-Only Design

All application access patterns use:

- GetItem
- Query

Table scans are avoided because they are slower and more expensive.

---

## Small Item Size

Store only required attributes.

Benefits:

- Lower storage
- Faster reads
- Faster writes

---

## Efficient Logging

Capture operationally useful information while avoiding verbose logs in production.

# Summary

The proposed architecture aligns costs with actual usage through a fully managed serverless design.

Key cost-saving characteristics include:

- No always-on servers
- Automatic scaling
- DynamoDB On-Demand billing
- Automatic TTL cleanup
- Minimal Global Secondary Indexes
- Query-first data model

These design decisions enable the solution to support unpredictable football ticket launches while maintaining operational efficiency and minimizing infrastructure management effort.
